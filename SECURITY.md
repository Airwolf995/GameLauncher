# Security Policy

## Supported Versions

Aktiv gepflegt wird jeweils der aktuelle Hauptstand auf `main`.

## Meldung von Sicherheitsproblemen

Bitte keine Sicherheitsluecken oeffentlich als GitHub Issue posten.

Melde Sicherheitsprobleme verantwortungsvoll mit:

- kurzer Beschreibung des Problems
- betroffenen Versionen oder Commits
- Schritten zur Reproduktion
- moeglicher Auswirkung

Bis ein eigener privater Meldeweg eingerichtet ist, sollten sicherheitsrelevante Details zunaechst vertraulich mit dem Maintainer abgestimmt werden.
